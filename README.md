# 🎲 game-nights

**Config-driven, single-file trivia boards that run in any browser.**
A quiz-show-style game engine where creating a new trivia night means copying a
folder and editing **one config file** — no build step, no framework, no server
required. Runs from a plain `file://`, from `python -m http.server`, and from
GitHub Pages.

Built for real office trivia nights and open-sourced so anyone can host their own.

- **Quiz board** — a categories × values grid; click a square to open a clue.
- **Automatic timer** — a car drives across the screen: direct answer clock, then
  an automatic steal clock.
- **± scoring** — full points for a correct answer, a half-value penalty for a
  wrong steal, positive-only bonuses; click a score to type an exact value.
- **Media** — answer-reveal images, embedded YouTube answer videos, and a music
  round (local mp3 clips or YouTube snippets).
- **Three themes** — Stadium Noir / Dusk / Day, switchable live.
- **Winner celebration** — confetti + an embedded victory song.
- **Printable team cards** — table tents and hand-out slips from the same roster.
- Everything (scores, played squares, teams, theme) auto-saves in the browser.

## Quick start

No install. Pick any of these:

```bash
# 1. Just open a board directly — double-click, or:
open events/demo/index.html            # macOS   (xdg-open on Linux, start on Windows)

# 2. Or serve the repo (some browsers restrict file:// for a few features):
python -m http.server 4173
#   → http://localhost:4173/            (the gallery)
#   → http://localhost:4173/events/demo/index.html

# 3. Or publish to GitHub Pages (see “Deploy” below) and share the link.
```

The root [`index.html`](index.html) is a **gallery** listing every event in
[`events.js`](events.js). Start with **[events/demo](events/demo/)** — it's the
fully-commented template.

│   │   ├── team-cards.html

An event's `index.html` is ~15 lines — it just loads the engine CSS, the config,
and the engine JS. All the markup is generated by the engine from your config.

## Make your own game

1. **Copy the template:** `events/demo` → `events/2026-09-quiz` (any slug you like).
2. **Edit `config.js`:** change `meta.slug` to match the folder, then rewrite the
   categories, teams, and media. `events/demo/config.js` documents every field.
3. **List it:** add one entry to [`events.js`](events.js) so it appears on the gallery.
4. Open `events/2026-09-quiz/index.html`. Done.

The only required config fields are `meta.slug`, `categories`, and `teams` —
everything else has a sensible default.

## Config reference

`config.js` sets one global, `window.TRIVIA_CONFIG`. Defaults are in `[brackets]`.

### `meta`
| field | notes |
|---|---|
| `slug` **(required)** | unique id; the localStorage namespace + gallery link |
| `version` `[1]` | bump to reset everyone's saved scores for this event |
| `pageTitle` `[title + titleAccent]` | browser tab text |
| `title` / `titleAccent` | brand — plain part + accent/bold part (e.g. `RVT` **`TRIVIA`**) |
| `logo` | optional brand mark shown top-left (local `assets/…` path or any URL) — drop in your own event/org logo; omit for a small accent dot |
| `author` | who made the board — credited on the rules card; mirror it in `events.js` so the gallery card credits them too |
| `subtitle` `[""]` | small line under the brand |
| `editionLine` `[""]` | shown on the rules + winner cards |
| `date` `[""]` | free-form label |
| `avTestUrl` `[rickroll]` | the “AV Test 🔊” button target (test the room's sound) |
| `legacyStorageKey` | optional — one-time migrate scores from a previous key |

### board & scoring
| field | notes |
|---|---|
| `timers` | `{ direct: 60, steal: 20, marker: "car" }` — answer clock, then auto steal clock (seconds); `marker` is the glyph that rides the timer bar (the built-in `"car"` SVG, or any emoji e.g. `"🏁"`) |
| `scoring` | `{ bonusValue: 100, stealPenaltyRatio: 0.5 }` — bonus points; wrong-steal costs this fraction of the value |
| `theme` | `{ default: "noir" }` — `noir` \| `dusk` \| `day` (press **Y** to cycle) |

### `categories`
An array of columns; each has a `cat` name and a `clues` array (rows, low→high value).
A clue's fields:

| field | notes |
|---|---|
| `v` | point value |
| `q` | the question/clue text |
| `a` | the answer |
| `clue` | host-only hint, revealed with **C** |
| `note` | host-only italic note under the question |
| `img` | answer-reveal image — a local `assets/…` path or any URL |
| `audio` | music round: a local clip, e.g. `"clips/song.mp3"` |
| `ytAudio` | `{ id, start, duration }` — play a YouTube audio snippet (fallback when the local clip is absent) |
| `yt` | `{ id, start }` — embed a YouTube **answer** video on reveal (supersedes `img`) |
| `b` / `bA` | a bonus question + answer (omit to have no bonus) |

### `teams` — one roster, shared by the board and the printable cards
```js
teams: {
  default: "A",                       // which preset loads first
  presets: [
    { key: "A", label: "Garage", confirmLabel: "Iconic Garage",
      teams: [
        { name: "Team R1T", color: "#F5C518", logo: "assets/r1t.png",
          tagline: "Four motors. Zero mercy." },   // tagline is used only by the cards
        // …
      ] },
  ],
}
```

### `celebrations`, `rules`, `winner`
```js
celebrations: {
  default: "daidai",
  songs: [ { key: "daidai", label: "Dai Dai", sub: "Shakira", yt: "…", start: 155, emoji: "🎉" } ],
},
rules: {
  openOnLoad: true,
  answerNote: { heading: "…", body: "…", example: "…" },  // omit → auto-generated
  items: [ { num: "1", text: "…" } ],                      // omit → auto-generated from timers
},
winner: { label: "🏆 Champions!!" },
```

> The in-game **Teams ⚙** drawer (**G**) also lets you switch the timer marker and
> nudge the winner-song **start** time live — handy for tuning at the venue. Those
> tweaks are saved in the browser and override the config values above.

## Text formatting (in `q` / `a` / `clue` / `b` / `bA`)

| you type | you get |
|---|---|
| `«word»`, `<<word>>`, `*word*` | a highlighted key term |
| `this` / `these` / `those` / `here` | auto-highlighted (question side only) — the classic quiz-show “pointer” words |
| `<em>…</em>`, `<b>…</b>`, `<br>` | inline HTML works |

The demo's **✍️ Formatting** category demonstrates each of these on screen.

## Host hotkeys

| key | action | | key | action |
|---|---|---|---|---|
| click square | open a clue | | **A** | reveal answer |
| **C** | reveal host hint | | **B** / **N** | reveal bonus Q / A |
| **1–6** | award points to a team | | **⇧1–6** | deduct |
| **Space** | pause/resume the timer | | **T** / **P** / **S** | restart direct / steal / stop timer |
| **M** | play/pause the music clip | | **Y** | cycle theme |
| **G** | teams & settings | | **W** | winner celebration |
| **?** | rules | | **Esc** | close |
| Shift-click a played square | reopen it as unplayed | | click a score | type an exact value |

## Music round: clips & the downloader

Each music clue can be answered two ways, and the engine prefers whichever is
available:

- **Local mp3** (`audio: "clips/song.mp3"`) — crisp, offline, no ads. Best at a
  live event.
- **YouTube snippet** (`ytAudio: { id, start, duration }`) — streams a few seconds
  from YouTube when no local file is present. This is what the public deploy uses
  (mp3s aren't committed — see below).

To build the local clips, describe them in the event's `songs.json` and run:

```bash
pip install "yt-dlp[default]" ffmpeg-downloader && ffdl install   # once
python tools/yt-download.py --event 2026-06-15-rvt-trivia         # downloads & trims
python tools/yt-download.py                                       # lists events with a songs.json
```

It downloads each clip's audio, trims it to `question_start … +question_duration`,
and drops it in the event's `clips/` folder with the filename `config.js` expects.
Existing files are skipped.

> ⚠️ **Ad caveat:** YouTube snippets from official/VEVO uploads may play an ad
> first without YouTube Premium or an ad-blocker — which is exactly why local
> mp3s are preferred at a real event. `ytAudio` is the zero-setup fallback for
> the public online board.

## Private content: the `config.local.js` overlay

Some clues are internal (inside jokes, colleagues' names) and shouldn't be public.
Put those in a **`config.local.js`** next to `config.js`. The engine loads it right
after `config.js`, so it can splice private categories into the board:

```js
// events/<slug>/config.local.js   — gitignored, never committed
(function () {
  var cfg = window.TRIVIA_CONFIG;
  cfg.categories.splice(2, 0, { cat: "🤫 Inside Jokes", clues: [ /* … */ ] });
})();
```

`config.local.js` is in `.gitignore`. A fresh clone simply doesn't have it — the
browser logs a harmless 404 and the public board loads without the private
category. This repo's June 2026 event uses exactly this pattern.

## Printable team cards

`events/<slug>/team-cards.html` reads the same `teams.presets` roster and renders:

- **Table tents** — one Letter page per team; fold along the centre line and stand
  it on the table.
- **Distribution slips** — N small copies per team; print, cut, shuffle, and hand
  one to each person so they find their team.

Print with the browser (**Ctrl/Cmd-P**) — `@page` rules size each tent to a full page.

## Deploy to GitHub Pages

The repo is already Pages-ready (there's a `.nojekyll` so `_`-prefixed paths and
all static files serve as-is):

1. Push to GitHub.
2. **Settings → Pages → Deploy from branch → `main` / root**.
3. Your gallery is live at `https://<user>.github.io/game-nights/`.

Link it from a Jekyll site (e.g. a `github.io` homepage):

```markdown
🎲 [Game Nights](https://<user>.github.io/game-nights/) —
   [June 2026](https://<user>.github.io/game-nights/events/2026-06-15-rvt-trivia/)
```

## Roadmap

Complexity gets added incrementally, only when a real game night needs it — the
default experience stays a folder + one config file.

- **View mode** — a mobile-friendly page for *participants*: follow the live board
  from your phone, see the current question, and buzz in.
- **Host mode** — run a specific *instance* of a board: its scores, played squares
  and remaining questions sync to everyone in view mode, so you can host without a
  shared TV screen.
- **Admin mode** — an in-browser builder that creates and edits the `config.js`
  (questions, teams, media) without touching code.
- **More game modes** — standalone music round, rapid-fire, final wager (the config
  already carries `meta.mode` for this).
